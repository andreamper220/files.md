# Структура жизни и артефакты

Система для фиксации **всего в жизни** — от проектов до отношений — через plain `.md` файлы и Telegram-бот.

## Иерархия

```
spheres/Work/              ← сфера (папка)
  Sphere.md                ← хаб сферы
  My Project/              ← проект (подпапка 1-го уровня в сфере)
    Project.md             ← хаб проекта
    drafts/                ← черновики
    final/                 ← финальные документы
    discussions/           ← обсуждения
```

**Правило:** любой документ (черновик, финальный, обсуждение) всегда лежит **внутри проекта**, а проект — **внутри сферы**.

`Life.md` — карта со ссылками на сферы, проекты и документы.

## Telegram-бот

### Первый запуск

`/life init` — создаёт `Life.md` и сферы: Work, Relationships, Health, Personal, Learning.

### Сохранение из чата

1. Отправь мысль в бот
2. **В черновик** / **Финализировать** / **В обсуждение**
3. Выбери **сферу** → **проект** (или пришли имя нового)

### Создание проекта

- `/life` → Сферы → сфера → **➕ Новый проект**
- Или при сохранении документа — пришли имя вместо выбора из списка

### Финализация

Черновик в проекте → открой файл → **✨ Финализировать** (переезд в `final/`).

### Перенос между сферами

У проекта или файла внутри `spheres/` — кнопка **↔️ В другую сферу**.

Переносится:
- весь проект (папка)
- отдельный `.md` файл

### Шорткаты

| Шорткат | Действие |
|---------|----------|
| `д Work/Проект текст` | черновик в проект |
| `ф Work/Проект текст` | сразу в финальные |
| `об Work/Проект текст` | в обсуждения |

Без `Сфера/Проект` — используется последний выбранный проект.

## Рабочий цикл

1. Сбросить мысль в Chat
2. Выбрать сферу и проект → черновик
3. Доработать в веб-приложении
4. **Финализировать** когда готово
5. При смене контекста — **перенести** проект в другую сферу
